# ¿Que es Git?
Git es un sistema de control de versiones distribuido que permite gestionar la evolución de un proyecto a lo largo del tiempo. Fue creado por Linus Torvalds en 2005 con el objetivo de ofrecer un sistema rápido, eficiente y seguro. Con Git, los desarrolladores pueden trabajar en equipo sin perder información, ya que cada cambio queda registrado en un historial. Además, Git permite crear ramas para desarrollar nuevas características sin afectar el código principal. Su modelo distribuido permite que cada clon del repositorio actúe como una copia completa del proyecto, garantizando seguridad y respaldo. Gracias a estas características, Git se ha convertido en la herramienta estándar en la industria del software.

# comandos de Git Explicaodos:

git init – Inicializa un repositorio en una carpeta local.
git add . – Agrega todos los archivos al área de preparación (staging).
git status – Muestra el estado actual del repositorio y los cambios realizados.
git commit -m "mensaje" – Guarda los cambios en el historial del repositorio.
git log – Muestra el historial de commits.
git branch – Lista, crea o elimina ramas.
git checkout rama – Cambia de una rama a otra.
git merge rama – Combina una rama con otra.
git remote add origin URL – Conecta el repositorio local con uno remoto.
git push – Envía los commits locales al repositorio remoto en GitHub.

# Flujo de trabajo recomendado
Actualizar el repositorio local antes de comenzar a trabajar.
Realizar los cambios necesarios en los archivos del proyecto.
Revisar qué archivos fueron modificados.
Preparar los cambios para guardarlos.
Registrar los cambios con un mensaje descriptivo.
Enviar los cambios al repositorio remoto en GitHub.
 Crear y usar ramas para trabajar nuevas funciones sin afectar la rama principal.
